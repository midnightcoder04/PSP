# Quickstart — RPC Tests + Security Audit

## Prerequisites

```bash
# .env.local must have all three (already required by db:seed)
VITE_SUPABASE_URL=https://okedskadkspeiyxjslqc.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=sb_publishable_…
SUPABASE_SECRET_KEY=sb_secret_…

# Plus one new env for the test fixtures (a single password reused across the 4 test users)
RPC_TEST_PASSWORD='choose-a-strong-throwaway-32-char-secret'
```

`RPC_TEST_PASSWORD` is loaded from `.env.local` via tsx's `--env-file` flag (matches the `db:seed` pattern). Choose any string ≥ 8 chars; it never touches production.

## Run the RPC contract suite

```bash
npm run test:rpc
```

What it does:
1. **beforeAll** (~3 s): cleans up any leftover `__rpc_test_*` rows; creates 4 users (admin / facilitator / participant / inactive); creates one session, one enrollment, two responses; signs each user in.
2. **describe blocks**: one per RPC, one per helper, plus `rls_matrix` and `edge_function_authz`.
3. **afterAll** (~2 s, best-effort): deletes the 4 users (CASCADE removes profile / enrollment / response rows) and the test session.

Expected runtime: **≤ 30 s** wall-clock against the hosted project (SC-001).

If `.env.local` is missing keys, the suite **auto-skips** with `it.skip` (matches `scripts/seed.test.ts`). `npm test -- --run` still exits 0.

## Run the security audit

```bash
npm run audit:security
```

Output: writes `specs/002-iter2-fixes/security-audit.md` with five sections:
1. **Supabase advisors** — table sourced from `mcp get_advisors`, with severity, ID, and current status (open / resolved-by-010 / accepted).
2. **RLS access matrix** — generated by issuing one SELECT per (table × role × op) tuple; each cell ✓ or ✗ with the policy name in parens.
3. **Edge Function authz** — invokes `create-user` as each role; reports the response code.
4. **Build sentinel scan** — runs `scripts/check-no-bypass.sh` (extended) and includes its output.
5. **Env-var policy** — greps `src/vite-env.d.ts` for `VITE_*SECRET*`; lists hits (should be zero).

The script always exits 0 (informational). The authoritative gate lives in `npm run test:rpc`'s `audit_assertions` describe block, which fails on:
- any open `0028` advisor on `get_admin_overview / get_session_stats / get_resume_position`
- any open `0029` advisor on `handle_new_user / update_progress_on_response`
- any RLS matrix cell that contradicts `contracts/helpers.md`
- any non-zero exit from `check-no-bypass.sh`
- any `VITE_*SECRET*` env declaration

## Run everything

```bash
npm test -- --run        # full vitest suite (105 → ~125 tests after this iter)
npm run audit:security    # writes the audit report
```

## Failure-mode playbook

| Symptom | Likely cause | Fix |
|---|---|---|
| `test:rpc` hangs at beforeAll | hosted project unreachable (network or paused) | check `mcp get_project status` is ACTIVE_HEALTHY |
| beforeAll fails with `Email already in use` | a previous run's afterAll failed; leftover `__rpc_test_*` users | run `npm run test:rpc` once more — beforeAll has best-effort pre-cleanup; or use Supabase Studio → Authentication → Users → filter `__rpc_test_` and delete manually |
| `get_admin_overview` test fails with `aggregate function calls cannot be nested` | migration 009 was rolled back or 006 was re-applied on top | re-apply 009 via `supabase db push` or MCP `apply_migration` |
| `is_admin` test for the inactive user passes (returns true) | someone changed migration 008 to drop the `is_active` clause | restore the `is_active = true` predicate in `is_admin` |
| `audit:security` reports a new advisor | a recent migration introduced a SECURITY DEFINER function without restrictive grants | extend migration 010 (or write 011) and re-run |

## Concurrency note

The fixtures use a fixed `__rpc_test_*` namespace. **Two concurrent runs will collide.** Run the suite serially.
