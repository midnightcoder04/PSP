**Branch**: `002-iter2-fixes` | **Date**: 2026-05-10 | **Spec**: `specs/002-iter2-fixes/spec.md`
**Input**: Feature specification from `/specs/002-iter2-fixes/spec.md`

## Summary

Land a vitest integration suite that calls every Postgres RPC and helper against the hosted Supabase project end-to-end (catching the nested-aggregate class of bug that bypassed our mocked unit tests in iteration 2), and a security audit pass that scrapes the Supabase advisors, derives an RLS access matrix from real role-bearing JWTs, audits the `create-user` Edge Function's authorization, extends the build-output sentinel script, and lands a hardening migration that resolves the open `0028`/`0029` advisor warnings.

## Technical Context

**Language/Version**: TypeScript 5.x on Node 20+ (vitest tests); deno runtime for the existing edge function
**Primary Dependencies**: vitest 1.x, @supabase/supabase-js 2.x — already in `package.json`. No new top-level deps.
**Storage**: Postgres 17 on Supabase project `okedskadkspeiyxjslqc` (region ap-south-1) — hosted only, no Docker.
**Testing**: vitest with `@vitest-environment node`; integration tests target hosted DB; auto-skip when env creds absent (matches `scripts/seed.test.ts` pattern).
**Target Platform**: macOS / Linux dev workstations with `.env.local` set; CI gate is manual (per Iteration 2 T124 disposition).
**Project Type**: Web SPA (Vite + React 18) with Supabase backend; this iteration is purely test + DB hardening — no UI changes.
**Performance Goals**: RPC suite ≤ 30 s wall-clock; existing 19-file 102-test suite must stay ≤ +1 s overall.
**Constraints**:
- Bundle budget: zero net bundle increase (no client-bundled code changes).
- No new top-level npm dependencies.
- Tests MUST NOT mutate or rely on the production admin user `1badbdb5-3615-459e-9810-0648fe338447`.
**Scale/Scope**: 11 functions in `public.*`, 7 RLS-protected tables, 1 edge function, 4 fixture roles (admin / facilitator / participant / inactive).
